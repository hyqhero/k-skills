# Vibe编程需求优化技能介绍

## 技能概述

面向编程小白的Vibe需求优化技能，帮助用户将模糊的想法转化为清晰的技术方案。通过多轮对话澄清需求，根据用户技术水平推荐合适的技术框架和数据存储方案，并生成规范的需求文档，避免因需求不清或技术栈选择错误导致的开发失败。

## 核心功能

1. **用户技术水平评估**：识别用户为完全小白/有基础/有经验三个等级，调整技术方案复杂度
2. **需求澄清**：多轮对话明确核心功能，避免过度设计
3. **智能技术栈推荐**：根据用户水平推荐差异化方案
   - 完全小白→多维表格（Notion/Airtable），避免数据库安装
   - 有基础→云数据库（Supabase），无需本地配置
   - 有经验→本地数据库（PostgreSQL），完全控制
4. **环境检查与安装指导**：自动检测操作系统（Windows/macOS/Linux），生成对应平台的安装命令
5. **需求文档生成**：输出结构化的Markdown文档，包含开发步骤和参考资料

## 设计思路

### 1. 以用户为中心的技术选型

传统技术选型往往基于"最佳实践"，忽略用户的实际能力。本技能的核心创新是**根据用户技术水平动态调整技术栈**：

- **完全小白**：优先无代码/低代码方案，用多维表格替代数据库
- **有基础用户**：推荐Supabase等云数据库，结合AI编程工具（Cursor/Copilot）
- **有经验用户**：提供完整技术栈选择，包含本地数据库和复杂框架

这样既降低了小白的入门门槛，又为有基础的用户提供成长空间。

### 2. 避免过度设计的MVP思维

编程小白最容易犯的错误是"想要一次做完美"。本技能强制引导用户：
- 聚焦1-2个核心功能
- 明确标注"暂时不需要"的功能
- 控制开发周期在1-2天内

确保第一个版本能快速跑起来，建立信心后再迭代。

### 3. 智能体与脚本的合理分工

遵循"实现方式选型指南"，充分利用智能体能力：

| 功能 | 实现方式 | 原因 |
|------|---------|------|
| 需求理解 | 智能体自然语言处理 | 需要对话和推理 |
| 技术框架对比 | 智能体领域知识 | 需要综合分析 |
| 需求文档生成 | 智能体内容创作 | 需要灵活填充 |
| 环境检查 | 脚本 | 需要执行系统命令 |
| 安装命令生成 | 脚本 | 基于模板的逻辑处理 |

这种分工既发挥智能体的语言能力，又利用脚本处理技术细节。

### 4. 跨平台适配的环境检查

环境检查脚本自动识别操作系统，生成针对性的安装命令：
- Windows：Chocolatey/Scoop/下载安装包
- macOS：Homebrew
- Linux：apt/yum/dnf/snap

避免用户因平台差异导致的安装失败。

### 5. 渐进式披露的资源组织

- SKILL.md：核心流程（213行，快速浏览）
- framework-guide.md：详细框架对比（529行，需要时查阅）
- requirement-template.md：文档模板（生成时参考）

减少上下文加载，按需读取。

## 使用流程

1. **评估技术水平**：询问编程经验，确定等级
2. **澄清需求**：多轮对话，明确核心功能
3. **推荐技术栈**：对比2个方案，结合用户水平推荐
4. **环境检查**：检查系统环境，生成安装命令
5. **生成文档**：输出完整的需求文档

## 技术亮点

- **零依赖**：脚本仅使用Python标准库，无需requirements.txt
- **轻量级**：16K打包文件，快速加载
- **可扩展**：framework-guide.md持续更新，支持新框架
- **AI友好**：针对有基础用户，推荐配合AI编程工具使用

## 适用场景

- 编程小白想要开发第一个项目
- 有基础但不确定技术选型
- 想快速验证想法的创业者
- 教学场景，指导学生完成项目

## 价值主张

让编程小白能独立完成从"想法"到"可执行方案"的转化，降低编程入门的技术门槛，保护学习信心，避免因技术选型错误导致的挫败感。
